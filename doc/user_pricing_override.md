# 用户专属价格配置说明

这份文档给运营和管理员使用。

## 这个功能解决什么问题

以前如果想给某个用户特殊价格，通常要新建很多分组。

例如：

```text
vip
vip_张三
vip_李四
vip_seedance优惠
```

分组会越来越多，后面很难维护。

现在可以不新建分组，直接给某个用户加一条“专属价格规则”。

一句话理解：

```text
用户原来能用什么不变，只是这个用户调用某些分组或模型时，价格单独算。
```

## 配置入口

后台页面：

```text
/console/setting?tab=ratio&ratio_tab=user_pricing_override
```

页面名称：

```text
用户价格覆盖
```

## 运营怎么配置

点击：

```text
新增用户价格规则
```

然后按 4 步配置。

### 第 1 步：选择你要做什么

页面里会先让你选择一个场景：

```text
给用户某个分组打折
给用户某个模型设置固定单价
给用户全部使用打折
给用户某个按量模型设置倍率
```

一般运营最常用的是前 3 个。

### 第 2 步：选择用户

搜索并选择用户。

下拉里会展示：

```text
用户 ID / 用户名 / 用户当前分组
```

例如：

```text
42 / Dexter / vip
```

选中用户后，系统会自动查出：

- 这个用户当前所在分组
- 这个用户可用的分组
- 这些分组下面的所有模型

### 第 3 步：选择生效范围

根据第 1 步选择的场景，页面会让你选择分组或模型。

#### 给用户某个分组打折

只需要选择分组。

例如：

```text
用户：42 / Dexter / vip
适用分组：sd2
倍率数值：0.8
```

效果：

```text
用户 42 使用 sd2 分组下的所有模型，按 8 折计费。
```

#### 给用户某个模型设置固定单价

选择分组，再选择模型。

例如：

```text
用户：42 / Dexter / vip
适用分组：grok
适用模型：grok-imagine-1.0-video
固定单价：0.3
```

效果：

```text
用户 42 使用 grok 分组下的 grok-imagine-1.0-video 时，按 0.3 固定单价计费。
```

#### 给用户全部使用打折

不需要选择分组，也不需要选择模型。

例如：

```text
用户：42 / Dexter / vip
倍率数值：0.9
```

效果：

```text
用户 42 所有使用都按 9 折计费。
```

#### 给用户某个按量模型设置倍率

这个主要给按 token 或按量计费的模型用。

例如：

```text
用户：42 / Dexter / vip
适用模型：gpt-4o-mini
倍率数值：0.5
```

效果：

```text
用户 42 使用 gpt-4o-mini 时，模型倍率按 0.5 计算。
```

如果不确定是不是按量模型，运营不要优先使用这个场景。

### 第 4 步：检查规则效果预览

弹窗底部会展示一段预览。

例如：

```text
用户价格规则预览：42 / Dexter 使用 sd2 / 全部模型 时，整体倍率使用 0.8。
```

确认预览符合预期后，再点确定。

最后点击：

```text
保存用户价格覆盖
```

## 数值怎么填

### 打折类

打折类填倍率。

常见值：

```text
1     原价
0.9   9 折
0.8   8 折
0.5   5 折
0     免费
```

### 固定单价类

固定单价会直接替换模型原来的固定价格。

例如：

```text
固定单价：0.3
```

表示该用户命中这条规则时，这个模型按 `0.3` 作为模型单价。

## 通配符怎么用

如果要对一类模型生效，可以手动填写模型通配符。

例如：

```text
seedance-*
doubao-seedance-*
grok-imagine-*
```

含义：

```text
seedance-* 匹配所有以 seedance- 开头的模型
```

运营如果不确定模型名，优先从模型下拉框里选择，不要手填。

## 优先级

规则越具体，优先级越高。

从高到低：

```text
用户 + 分组 + 模型
用户 + 模型
用户 + 分组
用户全部使用
系统原价
```

例如同时配置：

```text
用户 42 全部使用 9 折
用户 42 使用 sd2 分组 8 折
用户 42 使用 sd2 + seedance-2.0-480p 固定单价 0.25
```

当用户 42 调用：

```text
sd2 + seedance-2.0-480p
```

最终用最具体的规则：

```text
固定单价 0.25
```

## 推荐配置方式

### 常见需求 1：给大客户某个分组打折

选择：

```text
给用户某个分组打折
```

然后选择用户、分组，填倍率。

### 常见需求 2：给大客户某个视频模型固定价格

选择：

```text
给用户某个模型设置固定单价
```

然后选择用户、分组、模型，填固定单价。

### 常见需求 3：给某个用户所有使用打折

选择：

```text
给用户全部使用打折
```

然后选择用户，填倍率。

## 注意事项

- 这个功能只改价格，不改权限。
- 用户能不能用某个分组或模型，仍然由原来的用户分组、渠道和模型能力决定。
- 页面里选择用户后，会按这个用户当前权限加载可用分组和模型。
- 如果用户后续换了权限分组，计费时会按用户最新的真实分组匹配。
- 列表里的用户分组只是方便运营识别用户，不是最终权限判断。
- 如果不确定该选哪种场景，优先用：

```text
给用户某个分组打折
给用户某个模型设置固定单价
```

## 日志怎么看

如果请求命中了用户专属价格，管理员在日志详情里会看到：

```text
用户价格覆盖
```

里面会展示命中的分组、模型、价格动作和数值。

普通用户不会看到这些管理员计费细节。
